Description

**骨架优先设计（Skeleton-First Design）** 是一套语言无关的软件架构理论，旨在通过**四维坐标系统**严格定义代码的组织方式与依赖关系，从而构建高内聚、低耦合、可维护的系统。（侧重于工具库设计）。

该理论将每个代码单元映射到四个正交维度：

- **优先级层（priority）**：定义代码的抽象层级（core / framework / modules）及依赖方向。
- **能力领域（category）**：横向划分业务能力（如 audio、video、math）。
- **技术域（domain）**：标识代码依赖的外部世界（操作系统、第三方库、标准库等）。
- **结构模板（pattern）**：描述代码的角色与组织模式（如 interface、backend、utility）。

本文档是骨架优先设计的完整规范，包含核心思想、维度定义、依赖矩阵、语言适配指南、实施步骤及常见问题。无论您使用 C++、Java、Python、Rust 还是 Go，均可参照此理论组织项目。
