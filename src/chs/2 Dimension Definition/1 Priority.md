# 优先级层（priority）

`<priority>` 定义代码的稳定性等级、依赖风险、可移植性和**依赖方向**，是四维坐标中最重要的维度。
priority 越高，代表：
- 外部依赖越少
- 平台耦合越低
- 生命周期越长
- 复用性越强

priority 越低，代表：
- 依赖更多外部技术
- 变化频率更高
- 维护成本更高
- 更容易受生态影响
- 
## 取值
| 层级      | 含义               | 依赖规则                     |
|-----------|--------------------|------------------------------|
| core      | 最稳定，最少依赖        | 不依赖任何其他层             |
| framework | 中等稳定，单领域增强       | 只能依赖 core                |
| modules   | 依赖最多，变化最快       | 依赖 core + framework        |

## 判定
|外来生态使用数量| 层级 |
|-------------|--------------------|
| 0           | core        |
| 1           | framework   |
| N           | modules     |


## 铁律

- 依赖只能**向下**：modules → framework → core。
- **禁止环形依赖**：任何两个层之间不得相互依赖。
- core 层必须保持**技术中立**，不得引入任何具体第三方库或操作系统 API。
- framework 层负责提供某个领域的完整能力。单生态能力。
- modules 层负责跨领域集成。多生态协作能力。

## 示例

- `framework/video/ffmpeg/backend`：基于 FFmpeg 的具体实现。
